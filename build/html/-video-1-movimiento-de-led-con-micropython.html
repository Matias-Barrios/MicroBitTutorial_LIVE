Nuestro primer ejemplo: Moviendo un punto en la pantalla

¡Llegó el momento de ver la magia del código en acción! Para inaugurar nuestras prácticas textuales, vamos a analizar un programa clásico pero sumamente potente. Con este script vas a lograr algo fundamental en la computación física: interactuar con el hardware en tiempo real.

El objetivo de este primer ejemplo es controlar un único punto brillante de la matriz de luces LED de la micro:bit y desplazarlo horizontalmente de izquierda a derecha utilizando los botones físicos A y B.

Aquí tienes el código completo que introduciremos en el editor:

x = 2

y = 2

led.plot(x, y)


def moverAlaIzquierda():
    global x
    led.unplot(x, y)
    x -= 1 
    led.plot(x, y)

def moverAlaDerecha():
    global x
    led.unplot(x, y)
    x += 1
    led.plot(x, y)

input.on_button_pressed(Button.A, moverAlaIzquierda)
input.on_button_pressed(Button.B, moverAlaDerecha)

¿Por qué funciona este código?

Para entender el funcionamiento de este programa, debemos desglosarlo en cuatro conceptos clave que estructuran la lógica de la micro:bit:

El sistema de coordenadas y el estado inicial

La pantalla de la micro:bit es una matriz de 5 por 5 luces LED. En informática, las grillas se miden mediante coordenadas (X, Y), empezando a contar desde el número 0. Por lo tanto, tanto el eje horizontal (X) como el vertical (Y) van del 0 al 4.

Al declarar x = 2 e y = 2, estamos apuntando exactamente al centro geométrico de la pantalla. La instrucción inicial led.plot(x, y) le ordena a la placa encender (dibujar) el LED ubicado en esa posición. Al iniciar el simulador, verás un único punto brillante en el medio.

La ilusión de movimiento (Apagar, Modificar, Encender)

Si observas las funciones moverAlaIzquierda() y moverAlaDerecha(), ambas siguen una secuencia de tres pasos lógicos indispensables para simular que el punto se mueve:


Apagar (led.unplot): Antes de cambiar de posición, debemos apagar el LED actual. Si no hiciéramos esto, el punto dejaría un rastro continuo de luces encendidas a su paso en lugar de simular un desplazamiento.

Modificar la coordenada: Restamos uno (x -= 1) para ir a la izquierda o sumamos uno (x += 1) para ir a la derecha en el eje horizontal.

Encender (led.plot): Dibujamos el LED en la nueva coordenada de X que acabamos de calcular.
La importancia de la palabra clave global

Dentro de las funciones notarás la línea global x. En Python, las variables creadas fuera de las funciones son globales. Si una función intenta modificar su valor directamente, el lenguaje asume por seguridad que estás intentando crear una variable nueva y local, lo que rompería nuestra lógica. Al escribir global x, le estamos avisando explícitamente a la función que queremos alterar la variable X original que se encuentra afuera.

Programación orientada a eventos

Las dos últimas líneas son las encargadas de conectar el software con el mundo físico: input.on_button_pressed(). Esto le indica a la micro:bit que permanezca escuchando atentamente sus botones. Cuando detecta el evento de que el Botón A ha sido presionado, interrumpe lo que está haciendo y ejecuta de inmediato la función moverAlaIzquierda. Si presionas el Botón B, se dispara moverAlaDerecha.
